# Fraud Detection MLOps

> **Industrialization of a bank anomaly detection model (Credit Card Fraud)**  
> Stack: Python 3.11 · XGBoost · MLflow · FastAPI · Docker · GitHub Actions

---

## ⚡ Quick Start (Clean Machine)

```bash
# 1. Clone the repository
git clone <REPOSITORY_URL> && cd mlops-fraud-detection

# 2. Download the dataset
./download_data.sh

# 3. Start MLflow (UI available at http://localhost:5000)
docker compose -f docker/docker-compose.yml up mlflow -d

# 4. Run a complete training pipeline
docker compose -f docker/docker-compose.yml run --rm training

# 5. View results in the MLflow UI
open http://localhost:5000
```

## 🔧 Detailed Commands

### Local Environment (Without Docker)

```bash
# Install uv (fast Python package and project manager)
pip install uv

# Create the virtual environment and install dependencies
uv sync

# Run CI-3 (data quality gate)
uv run python -m src.data.quality_gate --data data/raw/creditcard.csv

# Run a training pipeline (local SQLite MLflow backend)
MLFLOW_TRACKING_URI=sqlite:///mlruns/mlflow.db \
  uv run python -m src.models.train --config configs/train_config.yaml

# View the runs leaderboard
uv run python -m src.models.compare --experiment fraud-detection-v1 --leaderboard

# Run unit tests (CI-1)
uv run pytest tests/ -v

# Check code formatting and linting (CI-2)
uv run black --check src/ tests/ api/
uv run flake8 src/ tests/ api/ --max-line-length=100
```

### Docker

```bash
# Build the training image
docker build -f docker/Dockerfile.train -t fraud-trainer:latest .

# Start MLflow standalone
docker compose -f docker/docker-compose.yml up mlflow -d

# Run training inside Docker
docker compose -f docker/docker-compose.yml run --rm training

# Start the prediction API (Bonus)
docker compose -f docker/docker-compose.yml --profile api up api -d

# Stop all services
docker compose -f docker/docker-compose.yml down
```

## 🔒 Security Architecture

- **Isolated Docker Network**: Internal bridge `fraud-net`, only ports 5000 and 8080 are exposed externally.
- **Non-root**: Containers execute under the unprivileged `appuser` user account.
- **Secrets Management**: No hardcoded credentials — relies on `.env` (gitignored) or CI/CD environment variables.
- **Data Privacy**: The `data/` directory is listed in `.gitignore` — raw banking data is never committed.
- **Minimal Base Images**: Uses `python:3.11-slim` to reduce the attack surface.
- **Deterministic Dependencies**: Use of `uv.lock` ensures exact reproducibility to avoid supply chain attacks.
- **Input Validation**: Pydantic strongly validates every API request (types, constraints).
- **API Security**: `API_SECRET_KEY` required via environment variable to secure the prediction endpoint.
